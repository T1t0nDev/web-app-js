2. Описание предметной области
Предметная область «Учёт клиентов сервиса» предназначена для регистрации, хранения и обработки информации о клиентах, которые пользуются услугами некоторого сервиса (например, сервисный центр, консалтинговая компания, интернет-провайдер и т.п.). Система позволяет вести базу клиентов с их контактными данными, а также отслеживать обращения (заявки) клиентов, их статусы и историю взаимодействия. Оператор системы может просматривать список клиентов и заявок, добавлять новые записи, изменять существующие и анализировать общее количество обращений. Решение ориентировано на учебное моделирование работы простой информационной системы без серверной части, с постепенным наращиванием функциональности в ходе выполнения лабораторных работ.

3. Пользователи системы
|Роль пользователя|Описание функций|
|-----------------|----------------|
|Оператор|Основной пользователь системы. Добавляет новых клиентов, просматривает список клиентов, редактирует их данные, работает с обращениями.|
|Администратор|(В будущих версиях) Расширенные права: удаление записей, управление пользователями, настройка системы.|

4. Основные сущности предметной области (данные)
|Сущность|Описание|Основные атрибуты|
|--------|--------|-----------------|
|Клиент|Физическое или юридическое лицо, пользующееся сервисом|ID клиента, ФИО / Название организации, Телефон, Email, Дата регистрации, Комментарий|
|Обращение (заявка)|Запрос клиента на оказание услуги или консультацию|ID обращения, ID клиента (связь с клиентом), Дата создания, Тема обращения, Описание, Статус (новая, в работе, выполнена, отменена)|

5. Пользовательские сценарии (Use Case)
|№|Сценарий|Краткое описание|
|-|--------|----------------|
|1|Добавление нового клиента|Оператор заполняет форму с данными клиента и сохраняет запись; система добавляет клиента в список.|
|2|Просмотр списка клиентов|Оператор открывает раздел «Список клиентов» и видит таблицу с сохранёнными клиентами и их основными атрибутами.|
|3|Редактирование данных клиента|Оператор выбирает клиента из списка, изменяет его данные в форме и сохраняет изменения; система обновляет запись.|
|4|Удаление клиента|(Для администратора) Оператор с соответствующими правами удаляет запись о клиенте после подтверждения действия.|
|5|Поиск клиента по имени или телефону|Оператор вводит текст в поле поиска, система фильтрует список и отображает только подходящих клиентов.|
|6|Создание обращения по клиенту|Оператор выбирает клиента и создаёт для него новое обращение, указывая тему и описание; система фиксирует обращение с датой и статусом «новая».|
|7|Изменение статуса обращения|Оператор открывает список обращений и переводит заявку в следующий статус.|
